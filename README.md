# Как мы разработали production Kubernetes-оператор за 7 дней вместо 90, используя LLM

Материалы к докладу на [DevOpsConf 2026](https://devopsconf.io/moscow/2026/abstracts/17475), 2 апреля 2026.

Михаил Петров, Yandex Cloud.

## О чём доклад

При разработке инфраструктурного кода для Kubernetes основная сложность — не написание отдельных фрагментов кода, а работа с большим контекстом: существующая архитектура, ограничения, обратная совместимость и риски изменений. Прямое использование LLM как «умного генератора кода» быстро упирается в ошибки и потерю контроля.

В докладе показан реальный кейс: разработка production Kubernetes-оператора за 7 дней вместо типичных 2–3 месяцев. Как LLM был встроен в инженерный процесс с декомпозицией задач, контекстом репозитория, верификацией решений и явными точками принятия решений человеком.
